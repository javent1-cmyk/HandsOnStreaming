# Spark Structured Streaming Assignment

## Overview

This project demonstrates Spark Structured Streaming using a simulated ride-sharing data stream. Streaming data is generated by a Python socket server and consumed by Spark for real-time processing and aggregation.

## Files

- `data_generator.py` - Generates simulated ride-sharing records in JSON format and sends them through a socket on localhost:9999.
- `task1.py` - Reads streaming data from the socket, parses JSON records, and displays the structured stream.
- `task2.py` - Calculates driver-level aggregations including total fare amount and average trip distance.
- `task3.py` - Performs event-time windowed aggregations with watermarking to calculate total fare amounts over sliding windows.

## Data Schema

Each generated ride record contains the following fields:

| Column | Description |
|----------|-------------|
| trip_id | Unique trip identifier |
| driver_id | Driver identifier |
| distance_km | Trip distance in kilometers |
| fare_amount | Trip fare amount |
| timestamp | Event timestamp |



## Running the Project

### Step 1: Start the Data Generator

Open a terminal and run:

```bash
python data_generator.py
```

The generator will continuously send JSON records to:

```text
localhost:9999
```

### Step 2: Run Task 1

Open a second terminal and run:

```bash
python task1.py
```

Task 1:

- Connects to the streaming socket
- Parses incoming JSON records
- Displays structured streaming data in the console

### Step 3: Run Task 2

```bash
python task2.py
```

Task 2:

- Groups records by driver
- Calculates total fare amount
- Calculates average distance traveled
- Writes results to CSV files

Output location:

```text
outputs/task_2/
```

### Step 4: Run Task 3

```bash
python task3.py
```

Task 3:

- Converts timestamps to event time
- Applies a 1-minute watermark
- Creates 5-minute windows with a 1-minute slide
- Calculates total fare amount per window
- Writes results to CSV files

Output location:

```text
outputs/task_3/
```

## Technologies Used

## Output

Results generated by Tasks 2 and 3 are stored in the `outputs` directory and demonstrate real-time aggregation of streaming ride-sharing data.
